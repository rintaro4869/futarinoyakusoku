# Apple Developer Program 承認後ロードマップ

> 「承認されました」のメールが届いたら、このドキュメントを上から順に進めてください。
> 全部で 3〜5 時間の作業です（Apple の審査待ち時間は除く）。

---

## フェーズ 1: ビルドを作る（30 分〜1 時間）

承認後、まず「アプリの本番ビルド」を作成します。

### やること

1. ターミナルを開く
2. 以下のコマンドを実行:
   ```
   EXPO_TOKEN=（あなたのトークン） npx eas-cli build --platform ios --profile production
   ```
3. 「Expo のサーバーでビルド中…」と表示されるので待つ（20〜40 分）
4. 完了メールが届いたら次のフェーズへ

### なぜこれが必要？
「ソースコード → iPhone で動くアプリファイル（IPA）」に変換する作業です。
EAS（Expo のクラウドサービス）がやってくれるので、Mac に Xcode は不要です。

---

## フェーズ 2: App Store Connect にアプリを登録する（30 分）

Apple の管理画面でアプリの「箱」を作ります。

1. https://appstoreconnect.apple.com を開く
2. 「マイ App」→「＋」→「新規 App」をクリック
3. 以下の通り入力:
   - プラットフォーム: iOS
   - 名前: `Pairlog — ふたりの記録`
   - プライマリ言語: 日本語
   - バンドル ID: `dev.pairlog.app`
   - SKU: `pairlog-v1`
4. 「作成」をクリック

---

## フェーズ 3: TestFlight でテストする（1〜2 時間）

本番ビルドが届いたら、提出前に実際に動かして確認します。

1. App Store Connect → 「TestFlight」タブ
2. ビルドが届いていたら「承認」をクリック（輸出コンプライアンスの質問に答える）
   - 暗号化の質問: **「はい、暗号化を使用しています」** → アルゴリズム: **「AES」のみ** → 「免除対象」を選択
3. TestFlight から自分の端末にインストール
4. `docs/testflight-checklist.md` を見ながら全項目を確認
5. 問題なければ次のフェーズへ

---

## フェーズ 4: App Store の情報を入力する（1 時間）

1. App Store Connect → 「App Store」タブ
2. `docs/appstore-text-draft.md` を開いて、各欄にコピペ
   - アプリ名 / サブタイトル / プロモーションテキスト / 説明文 / キーワード
3. スクリーンショットをアップロード（6.5" と 5.5" の 2 サイズ必須）
4. `docs/privacy-nutrition-label.md` を見ながら「App のプライバシー」を設定
5. プライバシーポリシー URL を入力（Notion の公開ページ URL）
6. 価格: 無料
7. カテゴリ: ライフスタイル / ソーシャルネットワーキング

---

## フェーズ 5: 審査に提出する（10 分）

1. 全セクションに緑のチェックマークが付いていることを確認
2. 「審査に提出」をクリック
3. 「App Review 情報」の「メモ」欄に `docs/appstore-text-draft.md` の審査メモを貼る
4. 提出完了！

### 審査の結果は？
- **承認（Approved）**: メールが届く。App Store に公開される
- **リジェクト（Rejected）**: メールに具体的な理由が書かれている。下の「リジェクト対応」を参照

---

## リジェクトへの対応方針

よくあるリジェクト理由と対応方法:

| リジェクト理由 | 対応方法 |
|---------------|---------|
| スクリーンショットが要件を満たしていない | 指定サイズで撮り直し |
| プライバシーポリシー URL がアクセスできない | Notion ページを「公開」に変更 |
| 審査担当者がログインできなかった | 審査メモにデモ手順を詳しく書き直す |
| アプリがクラッシュした | バグを修正して新しいビルドを提出 |
| 機能説明が実際と違う | 説明文を実際の機能に合わせて修正 |

リジェクトは珍しくありません。修正して再提出すれば OK です。

## 承認後のアップデート計画

公開後は以下の順番で機能を追加していきましょう。

### v1.1 — UI 改善（公開後 2〜4 週）
- スクリーンショットを実際の使用感に合わせて更新
- TestFlight フィードバックを反映したバグ修正
- ホーム画面のオンボーディング改善

### v1.2 — 通知強化（公開後 1〜2 ヶ月）
- リマインダー通知の細かな時間設定
- パートナーが記録したときのプッシュ通知
- 週次サマリーの通知

### v1.3 — ペア体験の深化（公開後 2〜3 ヶ月）
- カレンダービュー（記録の履歴を月表示で見る）
- 解放アクションのカスタマイズ機能
- ポイントしきい値の編集機能

### v2.0 — 収益化検討
- App Store のレビュー数が増えてから検討
- 広告なし。プレミアムプランか買い切りを想定

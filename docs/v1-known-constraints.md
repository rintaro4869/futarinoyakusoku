# 既知の制約・TODOメモ
**作成日**: 2026-03-20
**対象**: Pairlog v1

---

## 仕様上の制約（意図的）

### DBスキーマを変更しない
v1 再構築はフロントエンドのみの変更。バックエンド API・D1 スキーマはそのまま。
理由: Cloudflare Workers + D1 のマイグレーションリスクを避け、TestFlight → App Store 審査に集中するため。

### iOS 専用
Android 対応は v1.2 以降。
理由: Apple Developer アカウント取得後の早期リリースを優先。

### プッシュ通知はローカル通知のみ
FCM / APNs リモート通知は未実装。expo-notifications でのローカル通知のみ。
理由: バックエンドサーバー側の通知送信インフラがまだない。

## 既知の UI 制約

### タイムゾーン
`getTodayDayIndex()` はクライアントのローカル時刻に依存。
海外在住ユーザーで日付ズレが生じる可能性があるが、v1 スコープ外。

### カレンダーのパフォーマンス
月をまたぐたびに全ルールを再計算している。
ルール数が 100 件を超えると遅くなる可能性があるが、MVP 規模では問題なし。

### 記録のデバウンス
ホーム・約束タブ・カレンダータブそれぞれに独立した 3000ms デバウンスがある。
別タブで同一ルールを連打した場合は重複記録が起きる可能性（極低頻度・許容範囲）。

## 型安全性の警告

### expo-router の型
`router.navigate('/(home)/calendar' as any)` の `as any` キャスト:
expo-router v3 の TypeScript 型定義がタブルートの動的型付けに対応していないため。
ランタイムでは正常動作する。expo-router v4 へのアップグレード時に解消予定。
